## Methodology

- **Black-box**: this runner only calls `know-ctl query` via CLI. It does NOT re-implement or inspect the query/ranking logic.
- **Expected annotations** in scenarios.jsonl are based on user-value reasoning ("what should AI see when editing this file"), not on current algorithm behavior.
- **Special metric cases**:
  - empty expected + empty actual → P=100% R=100% F1=100 (correctly empty)
  - empty expected + non-empty actual → P=0% R=100% F1=0 (false positives)
  - non-empty expected + empty actual → P=100% R=0% F1=0 (complete miss)
