# know flow rules

<!-- 核心问题: 需求到哪了、验收标准是什么？ -->

## 1. 问题

write 流程有 8 个 step，每个 step 有多个判定点。审查发现 7 个盲区：hint 不匹配时的行为、非 roadmap 类型的版本判定、PRD 里程碑关联、参数修正后的重推断、update mode 无匹配 section、里程碑编号规则、文件冲突、索引版本 section 缺失、父文档不存在时的进度传播。每个盲区都可能导致 AI 自行决策产生错误。

## 2. 目标用户

使用 `/know write` 的开发者，期望流程行为确定性。

| 场景 | 痛点 |
|------|------|
| hint 输入了 "需求" | 不知道会匹配到 prd 还是报错 |
| 更新 ops 文档 | 不知道会新建 v2 还是原地更新 |
| 创建 PRD 但有多个里程碑 | 不知道 PRD 会关联到哪个里程碑 |

当前替代方案：遇到盲区时 AI 自行判断 — 不可预测，可能出错。

## 3. 核心假设

**每个判定点都有明确规则 → AI 在任何场景下的行为可预测，不再自行决策。**

验证方式：逐个判定点检查 write.md，每个都有 if/then 规则或 default 行为。

## 4. 方案

- **Before**: write 流程遇到未预见场景，AI 自行决策 → **After**: 每个判定点有明确规则，行为确定

### 补全的 7 条规则

| # | 判定点 | 补全规则 |
|---|--------|---------|
| 1 | 2a hint 不匹配 | 忽略 hint，fall back 到信号推断 |
| 2 | 2c 非 roadmap 版本判定 | 同 roadmap 规则：新方向 → 新版本，更新内容 → 原地更新。默认问用户 |
| 3 | 2d PRD 里程碑关联 | 从对话推断，歧义时问用户指定里程碑编号 |
| 4 | 3 参数修正重推断 | 改 type → 重推 name；改 name → 不重推 |
| 5 | 5 update 无匹配 section | 警告用户，选择新增内容或取消 |
| 6 | 5 里程碑编号 | 每个版本从 M1 开始，不续前版本编号 |
| 7 | 7 create 文件已存在 | 切换到 update mode |
| 附 | 8 索引版本 section 缺失 | 自动创建，插入在 Requirements 之前 |
| 附 | 8 父文档不存在 | 跳过进度传播，不创建父文档 |

### 任务

## 5. 验收标准

- write.md 的每个 Step 的每个判定点都有 if/then 规则或 default 行为
- 无"视情况而定"或未定义的分支
- 本次审查发现的 7 个盲区全部有对应规则

## 6. 排除项

- 不覆盖 learn/review 流程的判定点（各自流程文件负责）
- 不新增代码实现（纯文档规则补全）
