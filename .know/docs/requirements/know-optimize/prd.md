## 1. 问题

M1 提供了 6 个聚合指标，M2 提供了单条知识时间线，但用户看到"命中率 25%"后不知道该做什么。数据和行动之间没有闭环：不知道哪些指标有问题、问题意味着什么、该采取什么行动。每次看完 metrics 都需要人工判断，降低了可观测性的实际价值。

## 2. 目标用户

使用 know skill 的开发者，看到 metrics 数据后想知道下一步该做什么。

| 场景 | 痛点 |
|------|------|
| 命中率 25% | 不知道是该清理无用知识还是 scope 推断有问题 |
| 衰减率 40% | 不知道是存入质量低还是正常衰减 |
| 过期文档 3 个 | 不知道哪 3 个，要自己去 CLAUDE.md 找 |

当前替代方案：手动分析 metrics 输出 + 手动决定行动 — 需要理解每个指标的含义和阈值。

## 3. 核心假设

**metrics 附带优化建议 + review 智能排序 → 用户能在 2 分钟内完成一次有效的知识库优化。**

验证方式：用户按建议执行 review 后，再次运行 metrics 看到目标指标改善。

## 4. 方案

- **Before**: metrics 只显示数字，用户自己判断 → **After**: metrics 末尾附带针对性建议（哪个指标有问题 + 建议什么行动）
- **Before**: review 按 tier/age 排序，不区分优先级 → **After**: review 基于 metrics 数据智能排序（💤 条目优先、⚠ 条目优先）
- **Before**: 优化效果靠记忆对比 → **After**: 建议附带预期效果（"清理 3 条 hits=0 → 命中率预计从 25% → 50%"）

### 建议规则

| 指标 | 阈值 | 建议 |
|------|------|------|
| 命中率 < 50% | hits=0 条目占比 > 50% | 运行 `/know review` 清理 hits=0 的沉默条目 |
| 衰减率 > 30% | total_decayed / total_created | learn 存入质量需关注，检查 filter 规则是否过松 |
| 防御次数 = 0 | 无 active:defensive 命中 | 检查是否有 constraint 类知识，或 scope 推断是否匹配 |
| 覆盖率 < 50% | 多数 scope 未被查询 | 检查 recall 的 scope 推断规则是否覆盖项目结构 |
| 过期文档 > 0 | CLAUDE.md 有 ⚠ 标记 | 更新标记的文档 |
| 文档覆盖率 < 100% | 里程碑缺 PRD | 为未覆盖的里程碑创建 PRD |

无问题时 → `✅ 所有指标健康，无需操作`

### review 智能排序

基于生命阶段排序（替代原来的 tier/age 排序）：

1. ⚠ 濒危（即将被 decay）
2. 💤 沉默（hits=0 + 超 7 天）
3. 🆕 新建
4. ✅ 活跃

同阶段内按 age 降序（最老的优先）。

## 5. 验收标准

- `know-ctl metrics` 输出末尾 → 展示 0-6 条建议，每条包含指标名 + 问题描述 + 建议行动
- 所有指标健康时 → 展示 `✅ 所有指标健康，无需操作`
- 建议中涉及清理的 → 附带预期效果（"清理 N 条 → 命中率预计 X% → Y%"）
- `/know review` → 按生命阶段排序（濒危 > 沉默 > 新建 > 活跃）
- 用户按建议执行 review → 再次 metrics → 目标指标改善

## 6. 排除项

- 不支持自动执行建议（只建议，用户决定）
- 不支持自定义阈值（固定阈值，后续可调）
- 不支持跨指标关联分析（每个指标独立判断）
