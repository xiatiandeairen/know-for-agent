# M5 — 建立核心运营指标体系

## 1. 目标

### 完成定义

`know-ctl metrics` 命令可输出 learn/recall/write 三个维度的核心指标，数据来自真实使用而非模拟。

### 价值目标

用户可通过命令查看 learn/recall/write 各维度指标，快速判断知识库健康度和优化方向。

### 关键假设

- **假设**: 通过 learn/recall/write 三个维度的指标可以判断知识库健康度
- **类型**: 用户接受度
- **验证方式**: 查看 metrics 输出，判断能否回答"知识库是否健康、哪里需要优化"

## 2. 计划

### 预期交付物

- 代码: know-ctl metrics 命令 + metrics.json 聚合逻辑 + events.jsonl 事件记录
- 数据: 基于真实使用的指标数据

### 达标标准

| 维度 | 标准 |
|------|------|
| 功能完整性 | metrics 输出 learn/recall/write 各 2-3 个指标 |
| 质量 | 指标数据来自真实使用（events.jsonl），非模拟数据 |
| 性能与稳定性 | metrics 命令执行 <3s |
| 安全 | 本里程碑不涉及 |
| 可维护性 | 新增指标只需修改聚合逻辑，无破坏性变更 |

### 预估投入

1 天全职开发

### 决策路径

- **通过**: 进入 M6 生命周期追踪，指标体系为生命周期分析提供数据基础
- **未通过**: 调整指标定义，确保能反映知识库真实健康度

### 风险

依赖 events.jsonl 有足够真实事件积累；风险：早期使用数据量不足导致指标无参考价值
