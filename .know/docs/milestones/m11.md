# M11 — 知识库瘦身

## 1. 目标

### 完成定义

通过 review 清理无用条目，使知识库中每条知识都有存在价值，命中率从 1% 提升到 >30%

### 价值目标

用户的知识库不再被无用条目稀释，recall 查询时返回的都是有价值的知识

### 关键假设

- **假设**: 命中率低的主因是知识质量问题（大量无用条目稀释），而非匹配机制问题
- **类型**: 技术可行性
- **验证方式**: 运行 review 清理 hits=0 的条目后，重新统计命中率。如果命中率显著提升，说明问题在质量；如果不变，说明问题在匹配

## 2. 计划

### 预期交付物

- 数据: review 清理报告（删除/保留条目清单 + 理由）
- 数据: 清理后的 know-ctl metrics 快照（命中率基线）

### 达标标准

| 维度 | 标准 |
|------|------|
| 功能完整性 | review 完成，每条删除/保留有明确理由 |
| 质量 | 清理后命中率 >30%（know-ctl metrics 实测） |
| 性能与稳定性 | 本里程碑不涉及 |
| 安全 | 清理前备份 index.jsonl，可回滚 |
| 可维护性 | 建立定期 review 的触发条件（如命中率 <20% 时提醒） |

### 预估投入

0.5 天（review 操作 + 数据分析）

### 决策路径

- **通过**: 命中率 >30% → 进入 M12 优化 scope 匹配
- **未通过**: 命中率仍低 → 问题在匹配机制，跳过 M12 的前置分析直接进入 scope 优化

### 风险

风险：误删有价值但尚未被查询的新条目（缓解：只清理 >7 天且 hits=0 的条目）
