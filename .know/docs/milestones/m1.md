# M1 — 验证 learn 管线端到端可行性

## 1. 目标

### 完成定义

learn 管线从对话信号检测→候选生成→用户确认→写入 `.know/` 全链路可运行，无需手动干预中间步骤

### 价值目标

用户可从对话中自动提取知识并持久化到项目，不再需要手动维护知识条目

### 关键假设

- **假设**: AI 可以从对话中准确识别值得沉淀的隐性知识信号
- **类型**: 技术可行性
- **验证方式**: 在 know 项目自身的开发对话中运行 learn 管线，检查 10 次提取的准确率

## 2. 计划

### 预期交付物

- 代码: know-ctl append/query/hit/delete 命令
- 文档: learn.md workflow 文件（6 步管线定义）
- 数据: `.know/index.jsonl` + `entries/` 目录结构

### 达标标准

| 维度 | 标准 |
|------|------|
| 功能完整性 | 一条知识从对话检测→确认→写入，全链路无手动干预 |
| 质量 | 提取的知识准确反映对话内容，无捏造 |
| 性能与稳定性 | 单次 learn 操作 <30s，不阻塞对话 |
| 安全 | 本里程碑不涉及 |
| 可维护性 | know-ctl 命令接口稳定，workflow 文件可独立修改 |

### 预估投入

1 天全职开发

### 决策路径

- **通过**: 进入 M2 write 管线开发，learn 产出的知识库为 write 提供数据基础
- **未通过**: 排查 learn 管线阻塞点，暂不启动 write

### 风险

依赖: Claude Code plugin 机制稳定性；风险: AI 信号检测准确率未知，可能需要多轮迭代调整提取规则
