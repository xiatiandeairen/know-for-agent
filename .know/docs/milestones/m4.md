# M4 — 验证真实场景持续可用性

## 1. 目标

### 完成定义

在真实项目中连续使用 ≥1 周，日常开发流程不被 know 阻塞，知识库持续增长且有实际命中。

### 价值目标

用户可将 know 作为日常开发工具持续使用，而非一次性验证后搁置。

### 关键假设

- **假设**: know 的知识存取开销足够低，不会中断正常开发节奏
- **类型**: 用户接受度
- **验证方式**: 连续 7 天在 know 项目自身开发中使用，记录阻塞事件数量（0 次阻塞为通过）

## 2. 计划

### 预期交付物

- 数据: ≥10 条知识条目 + ≥1 次 recall 命中记录
- 文档: 使用过程中发现的问题清单（输入 v2/v3 规划）

### 达标标准

| 维度 | 标准 |
|------|------|
| 功能完整性 | 日常开发无阻塞，条目 ≥10，命中率 >0 |
| 质量 | 无误导性知识条目 |
| 性能与稳定性 | 连续使用 1 周无崩溃、无数据损坏 |
| 安全 | 本里程碑不涉及 |
| 可维护性 | 发现的问题可快速定位修复 |

### 预估投入

1 周持续使用观察

### 决策路径

- **通过**: v1 闭环验证完成，进入 v2 可观测性建设，优先解决"无法判断知识库健康度"的问题
- **未通过**: 如阻塞频繁，退回修复核心管线稳定性

### 风险

依赖: v1 核心管线（learn/recall/write）均已通过验证；风险: 长期使用可能暴露单次测试未触发的边缘情况
