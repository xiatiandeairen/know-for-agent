# M3 — 验证知识召回的实际防错效果

## 1. 目标

### 完成定义

agent 在新会话中修改代码前，自动命中已沉淀的相关知识，并因此避免了一次已知错误。

### 价值目标

用户不再因重复踩已知坑而浪费调试时间，agent 自动获得历史经验防护。

### 关键假设

- **假设**: 按 scope 匹配的知识召回可以在 agent 犯错前有效拦截
- **类型**: 技术可行性
- **验证方式**: 在新会话中触发修改 know-ctl.sh 的场景，观察 agent 是否命中 PATH 解析的 pitfall 知识并避免 dirname 解析 bug

## 2. 计划

### 预期交付物

- 文档: recall 机制定义（SKILL.md 中的 recall section）
- 数据: 至少 1 次真实防错记录（events.jsonl 中的 hit 事件）

### 达标标准

| 维度 | 标准 |
|------|------|
| 功能完整性 | agent 修改代码前自动命中相关知识 |
| 质量 | 命中的知识确实避免了已知错误（≥1 次真实防错） |
| 性能与稳定性 | recall 查询 <5s |
| 安全 | 本里程碑不涉及 |
| 可维护性 | 本里程碑不涉及 |

### 预估投入

0.5 天全职开发

### 决策路径

- **通过**: 进入 M4 持续可用性验证，核心闭环（learn→recall→防错）已跑通
- **未通过**: 分析 recall 匹配精度问题，调整 scope 匹配策略后重新验证

### 风险

依赖: learn 管线已产出足够知识条目供 recall 匹配；风险: scope 匹配粒度可能过粗或过细导致误命中/漏命中
