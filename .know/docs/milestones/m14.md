# M14 — recall 事件采集

## 1. 目标

### 完成定义

每次 recall 触发时自动记录 recall_query 事件到 events.jsonl，包含 scope、匹配数、展示数、action 类型

### 价值目标

用户可通过 events.jsonl 追踪 recall 的每次触发——什么时候查了什么、命中了几条、展示了几条

### 关键假设

- **假设**: 在 SKILL.md recall pipeline 中加一步事件记录，不会影响 recall 响应速度
- **类型**: 技术可行性
- **验证方式**: 添加 recall_query 事件后，recall 全流程耗时仍 <1s

## 2. 计划

### 预期交付物

- 代码: SKILL.md recall section 新增 Record query 步骤
- 代码: know-ctl.sh 新增 recall-log 子命令（或复用现有 event 写入）
- 数据: ≥5 条 recall_query 事件（实际使用产生）

### 达标标准

| 维度 | 标准 |
|------|------|
| 功能完整性 | recall_query 事件包含 ts/scope/matched/shown/action 5 个字段 |
| 质量 | 每次 recall 触发都有对应事件（0 遗漏） |
| 性能与稳定性 | recall 全流程耗时仍 <1s（事件写入 <10ms） |
| 安全 | 本里程碑不涉及 |
| 可维护性 | 事件格式与 events.jsonl 现有格式一致（append-only JSONL） |

### 风险

风险：事件写入失败不应阻塞 recall 主流程（缓解：写入失败静默跳过，不报错）
