# M15 — recall 数据消费

## 1. 目标

### 完成定义

know-ctl metrics 新增 recall 运行面板，展示触发/命中/空查分布和 scope 查询覆盖

### 价值目标

用户运行 metrics 命令后可直接看到 recall 的运行概况——不再是黑盒

### 关键假设

- **假设**: recall_query 事件数据足以支撑有价值的 metrics 面板
- **类型**: 用户接受度
- **验证方式**: 基于 ≥20 条 recall_query 事件，metrics 面板能回答"recall 触发了多少次、空查率多高、哪些 scope 是盲区"

## 2. 计划

### 预期交付物

- 代码: know-ctl metrics 扩展（recall 运行面板）
- 数据: 基于真实 recall_query 事件的 metrics 输出

### 达标标准

| 维度 | 标准 |
|------|------|
| 功能完整性 | metrics 新增：recall 触发次数、命中率（触发中命中的比例）、空查率、scope 查询覆盖 |
| 质量 | 指标基于 recall_query 事件计算，非估算 |
| 性能与稳定性 | metrics 命令执行仍 <3s |
| 安全 | 本里程碑不涉及 |
| 可维护性 | 新增指标只需修改 metrics 聚合逻辑 |

### 预估投入

0.5 天全职开发

### 决策路径

- **通过**: v5 完成，recall 全链路可观测。评估是否进入 v6（方向待定：多项目支持/社区推广/更多管线）
- **未通过**: 调整 metrics 面板展示内容，确保对用户有决策价值

### 风险

依赖: M14 recall_query 事件已上线且有足够数据；风险：数据量不足时 metrics 面板无参考价值（缓解：在 M14 上线后积累 1-2 周再做 M15）
