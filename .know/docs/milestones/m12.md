# M12 — recall scope 匹配优化

## 1. 目标

### 完成定义

recall 的 scope 推断规则优化后，覆盖率从 9% 提升到 >50%，agent 修改代码时能匹配到更多相关知识

### 价值目标

用户存的知识不再因为 scope 匹配过窄而被"遗忘"，recall 覆盖更多实际工作场景

### 关键假设

- **假设**: 当前 scope 推断规则过于依赖精确路径匹配，导致多数知识条目的 scope 与 agent 实际操作路径不匹配
- **类型**: 技术可行性
- **验证方式**: 分析 events.jsonl 中的 recall 查询记录，对比查询 scope 与知识条目 scope 的匹配率，识别匹配失败模式

## 2. 计划

### 预期交付物

- 文档: scope 匹配失败模式分析报告
- 代码: SKILL.md recall section 中优化后的 scope 推断规则
- 数据: 优化前后的覆盖率对比（know-ctl metrics）

### 达标标准

| 维度 | 标准 |
|------|------|
| 功能完整性 | recall 覆盖率 >50%（know-ctl metrics 实测） |
| 质量 | 新匹配不产生误命中（命中的知识确实与当前操作相关） |
| 性能与稳定性 | recall 查询耗时不因匹配范围扩大而超过 1s |
| 安全 | 本里程碑不涉及 |
| 可维护性 | scope 推断规则有明确的层级回退逻辑，新增 scope 模式只需添加映射规则 |

### 预估投入

1 天全职开发

### 决策路径

- **通过**: 覆盖率 >50% + 无误命中 → 进入 M13 量化 recall 价值
- **未通过**: 覆盖率仍低 → 分析是否需要引入语义匹配（当前为纯路径匹配），评估复杂度后决定

### 风险

依赖: M11 清理后的干净知识库；风险：放宽匹配可能引入误命中（噪声），需要平衡精度和覆盖率
