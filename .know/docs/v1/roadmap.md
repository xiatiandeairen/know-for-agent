# know 产品路线图

<!-- 核心问题: 产品到哪了、下一步往哪走？ -->

## 1. 产品愿景

| | |
|---|---|
| **解决什么问题** | AI agent 跨会话丢失隐性知识和设计讨论结果，重复犯错 |
| **给谁用** | 使用 Claude Code 等 AI 编程工具的开发者 |
| **核心差异** | 知识与代码同源同版本、分层控制 token 成本、自动衰减保持新鲜 |

## 2. 版本规划

### v1 — 两条管线闭环


| 维度     | 标准                             |
| ------ | ------------------------------ |
| **功能** | learn（知识沉淀）+ write（文档输出）两条管线闭环 |
| **质量** | 真实项目跑通全流程，无阻塞性 bug             |
| **用户** | 作者本人日常使用，验证核心假设                |


#### 里程碑

| # | 里程碑 | 验证点 | 进度 | 需求 |
|---|--------|--------|------|------|
| M1 | **能存** — learn 管线端到端跑通 | 一条知识从对话检测 → 确认 → 写入 `.know/`，无手动干预 | 1/1 | [learn](../requirements/know-learn/prd.md) |
| M2 | **能写** — write 管线端到端跑通 | 一次对话讨论 → `/know write` → 生成完整文档 + 索引更新 | 1/1 | [write](../requirements/know-write/prd.md) |
| M3 | **能防** — 沉淀的知识实际阻止了一次重复犯错 | agent 在新会话中命中已有知识，避免了已知错误 | 1/1 | — |
| M4 | **能用** — 真实项目连续使用 ≥1 周 | 日常开发无阻塞，知识条目 ≥10 条，命中率 >0 | 1/1 | — |


## 3. 当前版本

### 包含

| 能力 | 解决什么问题 |
|------|-------------|
| **Learn** | agent 跨会话丢失隐性知识 → 对话中检测并持久化，下次会话可用 |
| **Write** | 讨论结果散落在对话中，会话结束即丢失 → 按模板生成结构化文档，与代码同版本 |

### 排除

- 文档联动更新（级联一致性检查）
- 自定义模板
- 跨项目知识复用

## 4. 风险与依赖


| 风险     | 影响                      | 缓解           |
| ------ | ----------------------- | ------------ |
| 知识检索噪音 | 加载无关知识浪费 token、干扰 agent | 分层 + 衰减控制知识量 |
| 过度沉淀   | 用户疲于确认，learn 变成负担       | 路由拦截过滤低价值信息  |
| 模板适用性  | 文档结构不匹配实际场景             | 真实项目中迭代调整    |
