# know-for-agent Roadmap

## 产品定位

AI agent 的项目知识编译器。解决三个核心痛点：
1. **上下文丢失** — agent 每次从零开始，know 让它带着项目知识进入对话
2. **重复犯错** — agent 跨会话犯同样的错，know 记录隐性知识纠正心智模型
3. **设计结果散落** — 讨论完的设计决策留在对话里会丢，know 把它写成持久化文档

## 核心能力

| 能力 | 状态 | 说明 |
|------|------|------|
| Retrieve | ✅ | 按 scope 检索知识，静默注入 agent 上下文 |
| Learn | ✅ | 从对话中捕获高价值隐性知识，持久化到 .knowledge/ |
| Write | 🔧 | 将对话中的设计讨论结果结构化写入文档 |

## 投入与成本

| 项目 | 成本 | 说明 |
|------|------|------|
| 开发时间 | 个人业余 | 无固定 deadline，按优先级推进 |
| AI API 调用 | 包含在 Claude Code 订阅内 | know 本身不额外调用 API |
| 基础设施 | 零 | 纯本地文件，无服务端 |

整体成本：纯时间投入，无资金成本。

## Now

- **`/know write` 工作流** — AI 从对话推断文档类型，按模板写入 docs/
- **文档模板系统** — 9 种模板：roadmap / prd / tech / ui / arch / api / decision / ops / marketing
- **版本机制** — v1/v2/v3 递增，全量写入，旧版本保留
- **CLAUDE.md 文档索引** — 层级关系追踪（roadmap → prd → tech/ui/api）

## Next

- **write + retrieve 联动** — 写入的文档自动成为 retrieve 检索源
- **文档检索复用 CLAUDE.md** — 不单独做 read 管线，直接通过索引定位文档
- **模板迭代** — 根据实际使用反馈调整模板结构

## Later

- **文档联动更新** — 写入文档后检查关联文档状态一致性，提示用户是否级联更新（方向：写入后提示，用户确认才改）
- **自定义模板** — 用户在项目中添加自己的文档模板
- **文档关联链路检索** — 沿 roadmap→prd→tech 链路批量拉取上下文
- **跨项目知识复用** — 通用知识在多个项目间共享
