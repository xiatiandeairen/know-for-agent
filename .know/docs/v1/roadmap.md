# know Roadmap

## 产品愿景

AI agent 的项目知识编译器。让隐性知识（为什么选 A 不选 B、已知的坑、业务规则）持久化在代码仓库中，agent 跨会话不再重复犯错、不再丢失设计讨论结果。

目标用户：使用 Claude Code 等 AI 编程工具的开发者。差异化：知识与代码同源同版本、分层控制 token 成本、自动衰减保持新鲜。

## 版本规划

### v1 — 两条管线闭环
- **功能**: learn + write 两条核心管线完成，9 种文档模板可用
- **质量**: 在真实项目中跑通全流程，无阻塞性 bug
- **用户**: 作者本人日常使用，验证核心价值

## 当前版本

### 包含
- **Learn**: 8 步管线（信号检测 → 提取 → 路由拦截 → 层级评估 → 条目生成 → 冲突检测 → 确认 → 写入），6 种隐式信号、5 条 DROP 规则、2 问层级评估（重要/备忘）、2 阶段冲突检测，写入 `.knowledge/`
- **Write**: 9 步管线（触发 → 推断参数 → 确认 → 加载模板 → 填充 → 预览 → 写入 → 更新索引 → 完成），9 种文档模板，3 层目录（项目版本级 `v{n}/`、需求级、功能级），CLAUDE.md 索引，写入 `.know/docs/`
- **know-ctl.sh**: 7 个 CLI 命令（init / query / search / append / hit / decay / stats）

### 排除
- 文档联动更新（级联一致性检查）
- 自定义模板
- 跨项目知识复用

### 交付标准
- learn 和 write 各跑通一个完整用例
- 文档模板覆盖 9 种类型，实际生成 ≥3 份文档
- know-ctl.sh 所有命令可正常执行
- CLAUDE.md 索引正确维护 `#### v1` 和 `#### Requirements` 结构

## 风险与依赖

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| Skill 缓存热加载 | SKILL.md 更新后路由未生效 | 验证插件系统行为，必要时重启 |
| JSONL 规模增长 | 百条以上 jq 过滤性能下降 | 当前规模足够；必要时换存储 |
| 模板适用性 | 模板结构不匹配实际场景 | 在真实项目中迭代调整 |

## 协作要求

个人项目，作者兼任产品和开发。AI agent 负责执行实现，关键决策由作者确认。
