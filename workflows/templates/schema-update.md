# 接口规范 更新规则

## 变更类型

| 类型 | 含义 |
|------|------|
| **不可变** | 一旦写入不再修改 |
| **追加** | 只能新增条目，不能改已有的 |
| **可更新** | 内容可修改，但有约束 |

## 概览

| 位置 | 字段 | 变更类型 |
|------|------|---------|
| §1 概述 | 范围 | 不可变 |
| | 调用方 | 不可变 |
| | 协议类型 | 不可变 |
| §2 数据结构 | 整行 | 追加 |
| | 说明 | 可更新 |
| §3 接口定义 | 整个接口 | 追加 |
| | 已有接口.参数 | 追加（仅 optional） |
| | 已有接口.错误码 | 追加 |
| §4 约束与规则 | 约束条目 | 追加 |
| §5 示例 | 请求/响应 | 可更新 |

## 字段变更规则

### §1 概述

#### 范围 / 调用方 / 协议类型

- **变更类型**: 不可变
- **允许**: —
- **禁止**: 需求确认后修改（概述变更意味着这是一套新接口，应新建 schema 文档）
- **触发**: —
- **校验**: diff 中不应出现 §1 任何字段变化
- ❌ 把协议类型从 CLI 改为 REST
- ✅ 新建独立的 schema 文档描述新协议接口

### §2 数据结构

#### 整行（新增字段）

- **变更类型**: 追加
- **允许**: 新增字段行
- **禁止**: 删除已有字段（breaking change）；修改已有字段的类型/必填性
- **触发**: 接口扩展需要新字段
- **校验**: 新行满足 checklist 约束（类型具体、必填=是/否）
- ❌ 删除已有字段；将已有字段 类型 string→int
- ✅ 新增一行 optional 字段

#### 说明

- **变更类型**: 可更新
- **允许**: 说明文字更精确、补充边界条件
- **禁止**: 改变字段语义（说明变更不能导致调用方理解变化）
- **触发**: 发现说明不够清晰
- **校验**: 更新前后语义一致
- ❌ "用户 ID" → "组织 ID"（语义变更）
- ✅ "用户 ID" → "用户唯一标识，UUID v4 格式"（精确化）

### §3 接口定义

#### 整个接口（新增）

- **变更类型**: 追加
- **允许**: 新增完整接口定义（方法 + 路径 + 参数 + 响应 + 错误码）
- **禁止**: 删除已有接口（breaking change）
- **触发**: 产品需要新接口
- **校验**: 新接口满足 checklist 所有约束
- ❌ 删除已废弃的接口
- ✅ 新增一个完整的接口定义章节

#### 已有接口.参数

- **变更类型**: 追加（仅 optional）
- **允许**: 新增 optional 参数（必填=否）
- **禁止**: 新增 required 参数（breaking change）；删除已有参数；修改已有参数类型
- **触发**: 接口功能扩展
- **校验**: 新参数 必填=否
- ❌ 新增 必填=是 的参数
- ✅ 新增 必填=否 的 filter 参数

#### 已有接口.错误码

- **变更类型**: 追加
- **允许**: 新增错误码行
- **禁止**: 删除已有错误码；修改已有错误码含义
- **触发**: 发现新的错误场景
- **校验**: 新行满足格式约束
- ❌ 删除已有错误码
- ✅ 新增 "409 | 资源冲突 | 使用条件更新或重试"

### §4 约束与规则

#### 约束条目

- **变更类型**: 追加
- **允许**: 新增约束条目
- **禁止**: 删除已有约束；放宽已有约束
- **触发**: 发现新的边界条件或兼容性要求
- **校验**: 新条目具体可验证
- ❌ 删除频率限制约束
- ✅ 新增 "单次批量操作上限 100 条"

### §5 示例

#### 请求/响应

- **变更类型**: 可更新
- **允许**: 更新示例使其反映当前接口状态；新增示例
- **禁止**: 示例与 §2/§3 定义矛盾
- **触发**: 接口变更后示例需同步
- **校验**: 示例中的字段/类型/错误码与 §2/§3 一致
- ❌ 示例中使用 §2 中不存在的字段
- ✅ 接口新增参数后，更新示例展示新参数用法

## 操作流程

### 新增字段

1. §2 数据结构追加新行
2. §3 相关接口参数表追加（如需要，仅 optional）
3. §5 示例更新以反映新字段

### 新增接口

1. §3 新增完整接口定义章节
2. §4 追加相关约束（如需要）
3. §5 新增该接口的请求/响应示例

### 接口扩展（已有接口加参数）

1. §3 已有接口参数表追加 optional 参数
2. §3 已有接口错误码表追加（如需要）
3. §5 示例更新

## 校验规则

1. **§1 概述不可变** — diff 中不应出现范围/调用方/协议类型的变化
2. **字段只增不删** — §2 行数只增不减，已有字段类型/必填性不可变
3. **新参数必须 optional** — §3 已有接口新增参数 必填=否
4. **接口只增不删** — §3 接口数只增不减
5. **约束只增不减** — §4 条目数只增不减，已有约束不可放宽
6. **示例与定义一致** — §5 示例中的字段/类型/错误码与 §2/§3 匹配
