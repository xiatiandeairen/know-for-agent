# 能力全景 检查清单

## 概览

| 位置 | 字段 | 可省略 |
|------|------|--------|
| §1 能力清单 | 能力 | 不可（≥3 行） |
| | 描述 | 不可 |
| | 状态 | 不可 |
| | 版本 | 不可 |
| §2 覆盖范围 | 已知限制 | 不可（≥2 条） |
| | 未覆盖场景 | 不可（≥2 条） |

## 字段定义

### §1 能力清单

#### 能力

- **信息**: 用户可感知的能力名称
- **格式**: 用户视角的动作/功能名，非技术模块名
- **禁止**: 内部模块名（"数据处理模块"）；技术术语堆砌；实现细节
- **省略**: 不可
- **数据**: —
- ❌ "数据处理模块"
- ✅ "批量导入数据"

#### 描述

- **信息**: 该能力做什么
- **格式**: 1 句话，用户视角，说明能力带来的效果
- **禁止**: 技术实现描述；多句话；内部架构术语
- **省略**: 不可
- **数据**: —
- ❌ "基于 streaming parser 实现增量处理 JSON 数据"
- ✅ "上传 CSV/JSON 文件后自动解析并写入项目"

#### 状态

- **信息**: 该能力的当前可用性
- **格式**: 枚举值：可用 | 实验 | 计划
- **禁止**: 自定义状态值（"开发中""测试中""beta"）
- **省略**: 不可
- **数据**: —
- ❌ "beta 测试中"
- ✅ "实验"

#### 版本

- **信息**: 该能力所属版本
- **格式**: v{n}，n 为正整数
- **禁止**: 无 v 前缀的版本号；语义版本号（v1.2.3）
- **省略**: 不可
- **数据**: —
- ❌ "1.0" / "v1.2.3"
- ✅ "v1"

### §2 覆盖范围

#### 已知限制

- **信息**: 产品能力的已知边界
- **格式**: "{限制}（{影响}）"，≥2 条
- **禁止**: 无影响说明的裸限制；技术债描述
- **省略**: 不可
- **数据**: —
- ❌ "不支持并发"
- ✅ "单次导入上限 1000 行（超出需分批操作）"

#### 未覆盖场景

- **信息**: 产品明确不覆盖的使用场景
- **格式**: "{场景}（{原因}）"，≥2 条
- **禁止**: 无原因的裸场景；未来计划（→ roadmap）
- **省略**: 不可
- **数据**: —
- ❌ "多租户"
- ✅ "跨组织数据共享（当前架构为单租户设计）"

## 数据置信规则

实测标来源 > 估算标依据 > 目标标"待验证" > 无数据标原因。禁止编造。
